# Личный Telegram-автоответчик

Управляющий бот на aiogram подключает один пользовательский Telegram-аккаунт
через Telethon и отвечает на входящие сообщения по настраиваемым триггерам.
Доступ к интерфейсу имеет только `ADMIN_ID`.

## Установка

1. Создайте виртуальное окружение и установите зависимости:

   ```powershell
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1
   pip install -r requirements.txt
   ```

2. Заполните `config.local.json`:

   - `BOT_TOKEN` — токен управляющего бота от BotFather;
   - `ADMIN_ID` — Telegram ID единственного владельца;
   - `API_ID` и `API_HASH` — данные приложения с my.telegram.org;
   - `DATA_DIR` — каталог локальной базы и Telethon-сессии.

   Вместо JSON можно использовать одноимённые переменные окружения.

3. Запустите:

   ```powershell
   python trigger.py
   ```

## Поведение

- Повторное добавление аккаунта запрещено, пока запись аккаунта существует.
- «Отключить» после подтверждения удаляет запись аккаунта и локальную сессию.
- «Переподключить» использует сохранённую сессию.
- Код входа вводится сообщением владельца. Код не сохраняется в базу и не
  логируется.
- Облачный пароль удаляется из чата сразу после получения. Telegram-бот всё
  равно получает его для выполнения входа, поэтому используйте только своего
  бота и собственный аккаунт.
- Регистр при сравнении не учитывается. Если совпали несколько триггеров,
  отвечает первый по ID.
- Каждый триггер работает только в добавленных к нему чатах. Чат добавляется
  по числовому Telegram ID и проверяется через подключённый аккаунт. Если у
  триггера нет чатов, он нигде не срабатывает. На один триггер можно назначить
  до 20 чатов.
- Cooldown применяется отдельно к каждому диалогу.
- Каналы не обрабатываются; поддерживаются личные сообщения и группы.

SQLite и Telethon-сессия исключены из Git.
